Greedy Algorithms

Greedy algorithms ek approach hai jisme local optimal choices ko select karke overall optimal solution ko achieve karne ki koshish ki jati hai. Yeh algorithms optimization problems ko solve karne ke liye use hote hain.

Example ke liye, neeche ek greedy algorithm ka illustration hai jo local optimal choices ko select kar raha hai: 

Illustration of a greedy algorithm making local optimal choices

Greedy algorithms ko use karne ka reason yeh hai ki yeh simple aur efficient hote hain, aur inhe tab use kiya jata hai jab local optimal choices se overall optimal solution mil sakta hai.

Greedy Operations

Greedy algorithms par kuch common operations perform kiye ja sakte hain:

  • Activity Selection: Maximum activities ko select karna jo ek room mein ho sakti hain.
  • Job Sequencing: Jobs ko sequence karna maximum profit ke liye.
  • Huffman Coding: Data compression ke liye optimal prefix codes generate karna.
  • Fractional Knapsack: Items ko fractional parts mein select karna maximum value ke liye.
  • Minimum Coins: Minimum number of coins ko use karna ek given amount ko form karne ke liye.

Example ke liye, ek greedy algorithm ka code jo fractional knapsack problem ko solve karta hai: 

struct Item {
    int value, weight;
};

bool cmp(struct Item a, struct Item b) {
    double r1 = (double)a.value / a.weight;
    double r2 = (double)b.value / b.weight;
    return r1 > r2;
}

double fractionalKnapsack(int W, struct Item arr[], int n) {
    sort(arr, arr + n, cmp);
    int curWeight = 0;
    double finalValue = 0.0;
    for (int i = 0; i < n; i++) {
        if (curWeight + arr[i].weight <= W) {
            curWeight += arr[i].weight;
            finalValue += arr[i].value;
        } else {
            int remain = W - curWeight;
            finalValue += arr[i].value * ((double)remain / arr[i].weight);
            break;
        }
    }
    return finalValue;
}
